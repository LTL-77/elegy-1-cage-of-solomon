# Elegy -1: Cage of Solomon

`Elegy -1: Cage of Solomon` is an open development repository for the playable demo of the game currently titled `Elegy -1: Cage of Solomon`.

This repository contains only the public-facing game project: Godot scenes, scripts, configuration, and assets that are safe to distribute. Private setting documents, full lore, unreleased narrative drafts, and other internal materials are intentionally kept outside this repository.

## Project Status

The project is currently in early pre-production / prototype transition.

The current development target is a small vertical-slice demo built around:

- strong narrative presentation
- chapter-based progression
- static scene exploration
- 2D turn-based combat
- a self-contained story arc around the `2439` Beiyao incident

## Planned Demo Scope

The first playable demo is intended to cover:

- title screen
- opening story setup
- dialogue presentation
- simple scene interaction
- basic turn-based combat
- one boss encounter
- save/load foundation

## Tech Stack

- Engine: Godot `4.6`
- Rendering: `GL Compatibility`

## Repository Boundaries

Included here:

- public game code
- public project structure
- public demo data and placeholder assets
- open documentation needed for contributors

Not included here:

- full worldbuilding archive
- unreleased main-story drafts
- private design notes
- assets or references without redistribution rights

## Development Notes

This repository is being structured so the game can be built incrementally as a narrative-heavy demo first, then expanded later.

The immediate priority is not feature breadth. The immediate priority is a stable vertical slice with a clear tone, one identifiable combat hook, and a complete short play session.

# Scope Cut Rules

If implementation pressure rises, cut content before cutting architecture.

## Keep

- one short complete story arc
- one stable battle loop
- one boss encounter
- one clean return-to-title flow

## Cut First

- extra playable characters
- relationship thresholds
- full equipment and crafting
- map complexity
- optional side content
- CG unlock flow
- elaborate save slot UI

## Cut Last

- dialogue readability
- battle clarity
- basic save safety
- project structure that supports expansion
